# Contributing to stacks-node-mainnet

Thank you for your interest in contributing to this project!

## How to Contribute

### Reporting Issues

If you encounter any issues while setting up or running the Stacks node, please open an issue with:

- A clear description of the problem
- Your operating system and version
- Steps to reproduce the issue
- Any relevant logs (use `docker-compose logs`)

### Submitting Pull Requests

1. Fork the repository
2. Create a new branch for your feature or fix
3. Make your changes
4. Test your changes on a fresh Debian 12 installation if possible
5. Submit a pull request with a clear description of your changes

### Code Style

- Use consistent indentation (2 spaces for shell scripts)
- Add comments for complex logic
- Test scripts before submitting

### Areas for Contribution

- Documentation improvements
- Support for additional Linux distributions
- Script improvements and error handling
- Docker configuration optimizations

## Questions?

Feel free to open an issue for any questions or discussions.
